Collections.SubCollection Class

Defined in: ../../../application/modules/Carbon/resources/js/collections/collection.subCollection.js:1
Module: Carbon

SubCollection A general-purpose Collection for dealing with nested Models

SubCollections can be assigned to properties of Models, i.e. Model._subCol (they will not be synced) SubCollections contain SubModels and proxy all changes to original Model, while still maintaining the built-in events i.e. If we make Model.domains a SubCollection, you can still listen for changes on 'change:domains'

Nested model structure must be: { _id: 1, _body: { key: value } }

You should be able to nest SubCollections and SubModels, but this hasn't been tested
